
Constructions — мод для Minecraft (Forge), добавляющий инструменты и игровые механики для работы со «структурами": их размещение, демонтаж, сохранение и управление повреждениями.

**Описание**
- Мод реализует систему структур с превью при размещении, авторизацией, здоровьем у структур и реакцией на взрывные воздействия.

**Ключевые возможности**
- **Молот-строитель (Builder Hammer)**: зарядный инструмент — удерживайте ПКМ для зарядки (~10 секунд / 200 тиков), отпустите для демонтажа структуры под прицелом и получения предметов.
- **Корректная сериализация структур**: после перезагрузки мира блоки структуры сохраняются и восстанавливаются корректно.
- **Проверки размещения**: исправлены правила для trapdoor/ladder, рамп и оконных решёток, а также для half_wall/window_frame — уменьшены ложные срабатывания и предотвращены регрессии.
- **Взрывчатка (3 типа)**: слабая, средняя, сильная — наносят постепенный урон структурам (не обязательно добивают с одного взрыва). Урон игроков от взрывчатки настраивается через конфиг.
- **Рейд-зона (Raid Block)**: детонация сильных взрывчаток запускает блокировку строительства в радиусе ~30 блоков на 2 минуты; игроки внутри видят полосу босса (индикатор таймера/опасности).
- **Активы и локализация**: добавлены новые иконки взрывчатки (bobovka, sachelka, c4) и файлы локализации `ru_ru.json` и `en_us.json`.

**Управление и использование**
- Использование `Builder Hammer`: зажмите ПКМ для зарядки, отпустите — молот демонтирует структуру под прицелом.
- Взрывчатка: размещается и активируется стандартными взаимодействиями предметов; при детонации учитывается прочность структуры.

**Конфигурация**
- Конфиг мода доступен через `ConstructionsConfig` (серверный конфиг). В нём можно настроить параметры урона для игроков и другие опции. Текущие игровые значения: рейд-радиус ~30, длительность рейда ~120 сек.

**Установка**
- Скомпилируйте мод командой:
```bash
./gradlew build
```
- Поместите полученный JAR в папку `mods` вашего клиента/сервера Forge (1.20.1+).

**Отладка и советы**
- Если иконка `auth_cabinet` выглядит неправильно (розово‑чёрный квадратик), выполните перезагрузку ресурсов в клиенте или убедитесь, что `assets/.../models/item/auth_cabinet_item.json` содержит корректный `parent` (в репозитории уже применено исправление).
- При крашах смотрите `run/crash-reports` и `logs/latest.log` для стек-трейсов; в последних правках исправлены NPE, связанные с проверками опор рамок.
