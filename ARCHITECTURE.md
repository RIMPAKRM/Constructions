# Constructions Mod - Архитектура и документация

## Общий обзор

Mод Constructions - это система строительства баз для сервера DayZ-стиля на Minecraft 1.20.1 Forge.
Основана на механике строительства из игры RUST с поддержкой режима Adventure и интеграцией с модом spawnchest.

## Основные концепции

### 1. Структуры (Structures)

Все строения в моде являются объектами класса `Structure` или его подклассов:

- **FoundationStructure** (5x5): Базовый фундамент с адаптацией под ландшафт
- **WallStructure** (5x4): Стена, может быть ориентирована в 4 стороны
- **DoorFrameStructure** (5x3): Рама для двери
- **RoofStructure**: Крыша/потолок
- **DoorStructure**: Дверь (деревянная/железная), требует авторизации
- **AuthCabinetStructure**: Шкаф авторизации
- **StorageChestStructure**: Сундук (5 дней жизни)
- **CampfireStructure**: Костёр (2 часа жизни)

### 2. Здоровье и урон

Каждая структура имеет:
- `maxHealth` - максимальное здоровье
- `currentHealth` - текущее здоровье
- Метод `takeDamage()` для применения урона
- Метод `isDestroyed()` для проверки разрушения

Урон может быть применён только через специальную взрывчатку.

### 3. Скрепление структур

Структуры могут быть скреплены друг с другом:
- `attachedStructures` - набор ID прикреплённых структур
- При разрушении основной структуры, все прикреплённые разрушаются
- Если структура имеет несколько точек опоры, разрушается только та, что потеряла опору

### 4. Авторизация

Шкаф авторизации управляет доступом:
- Радиус действия: 40 блоков
- Может быть только 1 в радиусе 40 блоков
- Авторизованные игроки могут открывать двери
- Сундуки открываются всеми (но не авторизованные не могут строить рядом)
- Без шкафа: структуры исчезают через 3 дня, сундуки через 5 дней, костры через 2 часа

### 5. Взрывчатка

3 типа взрывчатки:
- **Weak** (2 урона): Самодельная слабая
- **Medium** (5 урона): Самодельная обычная
- **Strong** (10 урона): Военная

Взрывчатка:
- Наносит урон только структуре, на которую прикреплена
- Также создаёт взрыв как динамит
- Урон игрокам только в радиусе 5 блоков и при видимости

### 6. Адаптация под ландшафт

Фундамент может автоматически подстраиваться под рельеф:
- Максимальная высота каждого столба: 3 блока
- Высота вычисляется для каждого угла фундамента
- Средние точки интерполируются
- Автоматически удаляет листву и деревянные заборы

## Игровые механики

### Размещение структуры

1. Игрок в режиме Adventure берёт в руки предмет структуры
2. На клиенте показывается голограмма превью (синяя - возможна, красная - нет)
3. Проверяются условия:
   - Расстояние ≥ 100 блоков от зон лута
   - Если есть auth cabinet в радиусе, игрок должен быть авторизован
   - Блоки не должны быть заняты
   - Структура не должна пересекаться с воздухом
4. При клике ПКМ структура размещается на сервере
5. Сервер синхронизирует изменение всем игрокам

### Удаление структуры

1. Игрок берёт молоток строителя
2. Кликает ЛКМ + Shift на свою структуру
3. Структура и все прикреплённые к ней разрушаются

### Урон структуре

1. Игрок прикрепляет взрывчатку к структуре (ПКМ)
2. Кликает на взрывчатку (разминирует)
3. Взрыв наносит урон структуре и окружающим

## Конфигурация

Сервер-конфиг:
```yaml
building:
  minDistanceFromLoot: 100    # Минимальное расстояние от лута
  maxFoundationHeight: 3      # Макс высота столбов фундамента

structures:
  unclaimedStructureLifetime: 72  # часов
  chestLifetime: 120               # часов
  campfireLifetime: 2              # часов
  authCabinetRadius: 40            # блоков

explosives:
  weakExplosiveDamage: 2.0
  mediumExplosiveDamage: 5.0
  strongExplosiveDamage: 10.0
  explosionDamageRadius: 5
```

## Сетевая синхронизация

Пакеты синхронизации между клиентом и сервером:

**Клиент → Сервер:**
- `PlaceStructurePacket` - Размещение структуры
- `RemoveStructurePacket` - Удаление структуры
- `AttachStructurePacket` - Скрепление структур

**Сервер → Клиент:**
- `SyncStructuresPacket` - Синхронизация всех структур
- `UpdateStructureHealthPacket` - Обновление здоровья структуры

## Тестирование

Проект включает JUnit тесты для:
- Создания и управления структурами
- Системы здоровья и урона
- Менеджера структур
- Логики скрепления

Запуск тестов:
```bash
gradle test
```
